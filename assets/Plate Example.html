3.3.4Add Conditional Logic Using JavaScript

In the previous step, we introduced JavaScript basics, created a new JavaScript file, added a variable, and logged the value of that variable to the console. In this step, we will describe control flow and add conditional logic to the prework-study-guide repo.

Logging the value of topics to the console was pretty cool, but this is only the beginning of what JavaScript allows us to do in the browser. To understand how to use JavaScript, we first need to understand the concept of control flow.

Control flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page. The following brief video will help us understand this concept:

To better understand control flow, let's look at our current script.js file, shown here:

var topics = "HTML, CSS, Git, JavaScript";
console.log(topics);

If we were to switch these statements so that console.log(topics) is first, we would see undefined in the console. This is because we define topics AFTER we tell the computer to log it in the console.

Go ahead and give it a try. Switch the statements so that script.js now looks like the following code:

console.log(topics);
var topics = "HTML, CSS, Git, JavaScript";

Now open index.html in the browser, open Chrome DevTools, and select the Console tab. You will see the following in the console:

This experiment clearly demonstrated that the order of code affects the outcome. JavaScript is read from top to bottom, and order counts. This means that we must be intentional about how we set up code in a JS file; we can’t just throw it all in there any which way!

Let's undo the changes that we just made to script.js before moving on. Change the code back so that it looks like the following:

var topics = "HTML, CSS, Git, JavaScript";
console.log(topics);

Control flow can be interrupted, however. We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all. In this step, we’ll reverse engineer code to set up a conditional statement that lets us alter the control flow so that code executes only if a certain condition is met.

Code Demo

There are a few ways to create conditional statements in JavaScript. In this code demo, we will focus on the if statement. An if statement is a way of introducing decision-making into our code. Just like its name sounds, when using the if statement, code will only execute if the statement meets a condition, or is true.

Believe it or not, you have already worked with if logic before. Remember in Module 01 when we used pseudocode to sort plain plates from fancy plates? You might remember that we used something similar to an if statement in that activity. Let’s review some of the pseudocode from that activity, shown here:

Conditional: If (a plate is fancy) {
   put it on the top shelf
 } else if (a plate is plain) {
   put it on the bottom shelf
 }  else if (a plate is cracked) {
   put it in the trash
 }

Here, we introduced certain conditions that had to be met in order for us to know what to do with a dish. If a plate was fancy, we put it on the top shelf. If it was plain, we put it on the bottom shelf. The following image shows these conditions in action:

To create an if statement in JavaScript, we use the same logic and similar syntax, shown in the following code example:

if (condition) {
 // block of code to be executed if condition is truthy
}

When the computer encounters an if statement in a JavaScript file, it will check to see if the condition that we provide is truthy. If it is, then it will do whatever action we ask it to do in between the curly brackets. If the condition isn’t true or has a value that isn’t truthy, then the computer will ignore whatever is in between the curly brackets and move on.

deep dive
deep dive

What exactly do we mean by “truthy” and “falsy” in Javascript? A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy. To learn more about truthy and falsy values, visit the MDN Web Docs on truthy Links to an external site..

Sometimes we want to provide more conditions than just if. In JavaScript, we can have more than one condition—just like we did in the dishes pseudocode—by using else if, as demonstrated in the following code example:

if (first condition) {
   // block of code to be executed if first condition is truthy
} else if (second condition) {
   // block of code to be executed if second condition is truthy
}

The computer will continue through all of the conditions that we list, evaluating whether or not a condition is truthy. Once it finds a condition that is truthy, it will run the code in the curly brackets associated with that condition, and then break out of the if statement.

The following is an example of an if statement:

if (0) {
   console.log("I'm 0!")
} else if (null) {
   console.log("I'm null!")
} else if (true) {
   console.log("I'm true!")
}

Which message would be logged to the console in this case? If you said, "I'm true!", you're correct! The 0 and null values are both falsy, so the computer won't execute the code between the proceeding curly brackets. When the computer gets to the truthy value (true), it executes the code in the following line.

Feel free to add this example to script.js to try it out. Experiment by changing the values of the conditions to see what happens. Once you're finished, delete the code from the script.js file and move on.

In the next section, we will add a conditional statement to script.js to interrupt the control flow so that different code executes depending on the condition that evaluates as truthy.

Let’s Code!

Now that you’ve walked through reverse engineering a conditional statement, let’s try applying a conditional statement to our project.

Complete the following steps to add an if statement to the script.js file:

  1. In VS Code, open the prework-study-guide directory, and then open the script.js file.

  2. Replace the existing code in the file with the following code, and save the file:

    var topic = "HTML";

if (topic === 'HTML') {
  console.log("Let's study HTML!");
} else if (topic === 'CSS') {
  console.log("Let's study CSS!");
} else if (topic === 'Git') {
  console.log("Let's study Git!");
} else if (topic === 'JavaScript') {
  console.log("Let's study JavaScript!");
} else {
  console.log('Please try again!');
}

    As you can see in the preceding code block, first we replaced the topics variable with a new variable called topic with a value of HTML. Then we set up an if statement with several conditions.

important

Notice that we are using === in each condition. The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal. The following image shows a visual representation of this concept:

We are checking to see if the topic variable exactly equals a certain value. It will then run whatever code block corresponds with the condition that the computer evaluates as truthy. If none of the conditions evaluate as truthy, 'Please try again!' will be logged to the console.

  1. Save the file.

  2. Let's try it out. What do you predict we will find when we open the console in the browser? Open index.html in the browser, open Chrome DevTools, and select the Console tab. You should see something like the following image in the console:

    If you guessed "Let's study HTML!" would be logged in the console, then you were correct! The value of the variable topic is "HTML", so the computer evaluated the condition topic === 'HTML' as truthy. As a result, the string “Let’s study HTML” was logged to the console.

  3. Try changing the value of topic to 'CSS' in the script.js file and then save the file. What do you think will happen?

    When you refresh the page in the browser, you should get the result shown in the following image:

    "Let's study CSS!" is now logged to the console because we changed the value of topic.

  4. Now try changing the value of topic to "Hello" or to a different value that isn't covered in our conditional statement.

    When you refresh the page in your browser, you should see something like the following image:

    None of the conditions evaluated as truthy. Therefore, the final code block in the conditional statement was executed.

Take a moment to experiment with the logic in script.js. When you are finished, make sure that you change the value of topic back to 'HTML' before moving on.

Remember, "Let's study HTML!" is being logged to the console because the value of topic is set to 'HTML'. When the computer checks the conditions in the if statement, topic === 'HTML' evaluates to truthy, and so the code block, console.log("Let's study HTML!") is executed.

Summary

Congratulations on reverse engineering an if statement! Remember, control flow is the order in which a computer executes code in a file. Typically, JavaScript code is executed from top to bottom. However, using conditional statements such as if statements allows us to interrupt control flow.

When creating an if statement, we provide one or more conditions for the computer to evaluate. If one condition is truthy, the computer will execute the code that we provide between curly brackets. But if the condition is not truthy, the code will not execute, interrupting the control flow. Interrupting the control flow in this way allows us to build apps that respond to a user's actions.

You may have noticed in this step that we had to manually change the value of topic if we wanted a different condition to evaluate as truthy. How would we do something like this if we had a large amount of data? JavaScript has a solution for this, which we will investigate in the next step.
